# 回忆与日历模块 (Memories & Calendar)

## 1. 这是用来干嘛的？
**目标**：给用户带来成就感，并制造意外的感动。
写日记如果只是单向输出，久而久之会觉得枯燥。这个模块的作用是把死板的文字变成“活的回忆”。一方面通过打卡图让用户看到自己的坚持，另一方面像时间胶囊一样在特定的日子里把过去的自己推送到眼前。

## 2. 核心逻辑与功能表现

### 2.1 日历打卡热力图 (成就感反馈)
- **场景**：用户想知道自己这个月写了多少篇日记，有没有坚持下来。
- **逻辑**：
  - 在 App 中提供一个单独的“日历”视图（或者是类似 GitHub 的全年热力图）。
  - 用户哪天写了日记，那一天在日历上就会有一个圆点标记（如果那天写得多，颜色还会更深）。
  - 当用户在日历上点击某一个有标记的日期时，下方会立刻列出那一天记录的所有日记卡片，方便快速阅读。
  - 这个功能直观地展示了用户的“足迹”，是极强的持续记录驱动力。

### 2.2 那年今日 (时间胶囊与感动)
- **场景**：今天正好是 6 月 25 日。用户一打开 App，突然看到首页最上方漂浮着一张小卡片：“2023年的今天，你写下了这段话...”。
- **逻辑**：
  - 每天打开 App 时，系统会在本地数据库里默默检索以往年份（一年前、两年前、三年前...）的今天。
  - 如果匹配到了日记，就会生成一张精美的卡片放在首页顶部。
  - 用户不需要费力去文件夹里翻找，系统主动把回忆“喂”到嘴边。点击卡片就能穿越时空，感受当时的喜怒哀乐。这是所有日记 App 中公认的最容易让用户产生情感粘性的功能。
