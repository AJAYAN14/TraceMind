# 地理位置记录 (Location Tagging)

## 1. 这是用来干嘛的？
**目标**：为记忆加上空间的锚点。
虽然我们暂时不开发宏大的“足迹地图”视图，但在写日记的那一瞬间，悄悄记录下用户身处何方，是非常有意义的。这让日记不只有时间属性，还有了空间属性。未来如果想做足迹回顾，这些数据就是现成的宝藏。

## 2. 核心逻辑与功能表现

### 2.1 自动获取与权限
- **场景**：用户开始写一篇新的游记。
- **逻辑**：
  - App 首次运行时，会向用户请求“仅在使用中允许”的定位权限。
  - 只要有权限，当用户新建日记时，系统会在后台静默获取一次当前的经纬度。
  - 如果用户拒绝了定位权限，这篇日记就不带有位置信息（不强制）。

### 2.2 逆地理编码（经纬度转文字）
- **场景**：用户看不懂经纬度，需要显示人类能看懂的地名。
- **逻辑**：
  - 获取到 `Latitude` (纬度) 和 `Longitude` (经度) 后，调用 Android 系统底层的 `Geocoder` 服务。
  - 它会把冷冰冰的坐标翻译成如“**北京市·朝阳区**”或具体的街道名。
  - 这行精简的文字，会显示在日记标题的下方，陪伴着天气和心情图标。

### 2.3 数据库字段扩充
- 为了支持这个功能，底层的 `Diary` 数据库表只需增加三个极小的字段：
  - `latitude: Double?` (纬度)
  - `longitude: Double?` (经度)
  - `addressName: String?` (具体地名文本)
- 这种方式几乎**完全不增加 App 的体积**，也不影响现有的备份方案。
