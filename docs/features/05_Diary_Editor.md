# 沉浸式图文日记编辑器 (Diary Editor)

## 1. 这是用来干嘛的？
**目标**：提供一个像 Word 一样好用，但又极具个性化和自由度的书写面板。
日记不仅是干瘪的文字，往往还伴随着照片。这个页面用来让用户记录灵感、贴上照片、选择好看的背景纸，完成一次有仪式感的记录。

## 2. 核心逻辑与功能表现

### 2.1 图文自由穿插混排
- **场景**：用户想写一段话，然后配一张刚刚拍的照片，接着在照片下面继续写字。而不是把所有文字写完，再把图片全部堆在底部。
- **逻辑**：
  - 用户正常打字。
  - 点击键盘上方的“相册”或“拍照”按钮。
  - 选中一张图后，图片会立刻插入到当前打字光标所在的位置。
  - 图片的下方会自动产生一个新的文本输入区，用户可以接着这幅图继续讲故事。
  - 这种逻辑被称为“块状编辑器”，文字和图片像积木一样上下堆叠，互不干扰，排版非常漂亮。

### 2.2 个性化背景更换
- **场景**：用户今天心情很好，想给这篇日记换一个明黄色的纸张背景，或者用一张美丽的星空图作为这篇日记的底板。
- **逻辑**：编辑器提供一个“更换背景”的入口，用户可以挑选系统预设的颜色，或者从相册选一张图。选好后，当前日记的背景立刻变幻，提升书写沉浸感。

### 2.3 自动记录与元数据
- **场景**：用户在记录时，系统应该比用户更贴心，主动记下此刻的时间和氛围。
- **逻辑**：
  - 打开编辑器的一瞬间，系统自动记录当前的日期和具体时间。
  - 提供快速选项，让用户点选今天的天气（晴、雨、雪）和此时的心情（开心、难过、平静）。

### 2.4 无感自动保存
- **场景**：用户写到一半突然有事退出了 App，或者手机没电了，辛辛苦苦写的内容不能丢。
- **逻辑**：废弃传统的“点击保存”按钮。用户在输入的过程中，只要退出当前界面、切回桌面，或者每隔几秒钟，系统就会在后台默默将文字和引用的图片信息保存进本地数据库。用户永远不用担心忘记保存。
