# 日记列表浏览模块 (Diary List)

## 1. 这是用来干嘛的？
**目标**：帮用户找回回忆。
当用户点击了首页或者进入了某个文件夹（比如“打工日记”）时，需要有一个页面来把过去写的日记一篇篇列出来，让用户能直观地看到过去的记录。它就像是一个目录大纲。

## 2. 核心逻辑与功能表现

### 2.1 瀑布流/卡片展示
- **场景**：用户想看看自己上周去旅游都写了啥。
- **逻辑**：
  - 日记被设计成一张张精美的卡片，从上到下按时间顺序排列（最新的在最上面）。
  - **视觉抓取**：如果日记里有照片，系统会自动抽取第一张照片作为这张卡片的“封面图”。这样用户不用点进去，扫一眼图片就能唤醒回忆。
  - **摘要显示**：卡片上会显示标题；如果没有写专门的标题，系统会自动截取正文的前两句话作为摘要显示出来。同时附带当天记录的天气和心情小图标。

### 2.2 搜索与检索
- **场景**：用户记得一年前写过一篇关于“面试”的日记，但是文件夹里有上百篇，实在找不到。
- **逻辑**：页面顶部有一个搜索框。用户输入关键词“面试”，系统会立刻筛选出所有标题或内容中包含“面试”这两个字的日记卡片，方便快速定位。

### 2.3 快捷新建入口
- **场景**：用户正看着“日语学习”文件夹里的旧日记，突然想立刻写下一篇新的日语笔记。
- **逻辑**：在这个列表页面的右下角，永远悬浮着一个显眼的“+（写日记）”按钮。用户点它新建的日记，会自动归类到当前所在的“日语学习”文件夹里，无需写完后再手动移动，省去归类的繁琐。

### 2.4 日记操作
- **场景**：用户发现有一篇日记归错类了，想把它移动到其他文件夹，或者觉得某篇写得不好想直接删掉。
- **逻辑**：长按某张日记卡片，会弹出一个菜单，提供“移动到其它文件夹”和“删除”的选项。确认删除后，该日记会从列表中彻底消失。
