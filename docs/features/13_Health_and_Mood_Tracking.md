# 个人状态与健康追踪 (Health & Mood Tracking)

## 1. 这是用来干嘛的？
**目标**：不仅记录“发生了什么”，更记录“身体和精神的状态”。
日记最深刻的价值在于帮助用户了解自己。有时候，用户的低落并不是因为发生了坏事，而是因为生理期（姨妈）或者持续的高压。把“心情”、“压力”、“生理期”和“性生活”作为标准化的标签记录下来，能帮助用户在日后复盘时，发现自己身体和情绪的隐藏规律。

## 2. 核心交互逻辑与 UI 表现

这四个维度的数据不能让用户手打文字去记录，那太繁琐了。必须在写日记的界面提供**“一键点击”**的快捷面板。

### 2.1 心情 (Mood)
- **逻辑**：采用经典的 5 级量表。
- **UI 表现**：在日记编辑界面的顶部，提供 5 个显眼的 Emoji（😭 糟糕、😔 低落、😐 平静、😊 开心、🤩 极好）。用户点一下即可选中。

### 2.2 压力指数 (Stress Level)
- **逻辑**：压力是一个线性的感知，不需要复杂的文字。
- **UI 表现**：提供一个“滑动条 (Slider)”，或者简单的 3 个档位按钮（🟢 轻松、🟡 中等、🔴 高压/内耗）。

### 2.3 生理期追踪 (Menstruation)
- **逻辑**：主要针对女性用户的核心生理周期记录。
- **UI 表现**：一个简单的开关按钮（🩸 姨妈期：开启/关闭）。如果是开启状态，可以额外折叠出一个小选项，选择流量（少/中/大）或痛经程度。

### 2.4 性生活 (Sex Life)
- **逻辑**：极其私密的个人生活记录。
- **UI 表现**：一个低调的开关按钮（❤️ 性生活：开启/关闭）。为了保护隐私，这个按钮在未开启时可以设计得非常隐蔽，或者允许用户在设置里选择是否要在记录面板中显示这个选项。

## 3. 数据存储逻辑 (Database Design)

为了不污染现有的日记正文，这四个维度的数据应该作为**结构化数据**存入本地 SQLite 数据库。

**方案 A（单独字段拓展）**：
直接在 `Diary` 实体表中增加这几个轻量级字段：
- `mood: Int?` (1-5 代表心情等级)
- `stressLevel: Int?` (1-3 代表压力等级)
- `hasPeriod: Boolean` (是否处于生理期)
- `hasSex: Boolean` (是否有性生活)

**方案 B（高扩展性 JSON 方案）**：
如果在未来您还想增加“睡眠质量”、“是否饮酒”、“是否运动”等无数个追踪项，最好在 `Diary` 表中加一个 `trackerData: String?` 字段，把这些指标打包成 JSON 存进去，这样以后增加任何新指标都不需要修改数据库表结构。

## 4. 终极杀手锏：数据交叉分析 (Data Insights)

如果用户坚持记录了几个月，这四个简单的数据将产生巨大的价值。在 App 的“统计分析”页面，系统可以自动得出这样的结论：

- **相关性分析**：“记录显示，每当处于 `🩸姨妈期` 时，你的 `压力指数` 会比平时高出 40%，且心情多为 `😔低落`。下个月的这几天，请对自己好一点。”
- **正反馈关联**：“数据表明，当记录了 `❤️性生活` 后的第二天，你的心情为 `🤩极好` 的概率高达 80%。”

这种**以数据反哺用户认知**的功能，能让您的日记 App 变得不仅是一个树洞，更是一个懂用户的贴身健康情绪管家。
