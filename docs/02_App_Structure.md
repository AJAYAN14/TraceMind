# 应用整体结构与导航逻辑

## 1. 这是干什么的？
本文档用于说明 TraceMind (迹忆) 整体的页面流转逻辑，以及各个大功能模块之间的关系。它的作用是让开发者和设计者明白：用户打开 App 后能看到什么，点击什么按钮会进入什么页面。

## 2. 核心模块与层级

### 2.1 首页（默认视图）
- **定位**：用户打开 App 后的第一眼看到的页面，也是整个目录树的“根目录”。
- **展示内容**：
  - 顶部的全局功能栏（例如：设置入口、全局搜索入口、日历视图入口）。
  - **那年今日**记忆卡片（如果过去同月同日写过日记，会浮在最上方供回顾）。
  - 根级别的**文件夹卡片**（例如用户创建的“学习”、“工作”、“生活”大分类）。
  - 根级别的**日记流**（没有被放进任何文件夹的散落日记）。
- **主要操作**：可以点击“新建文件夹”，也可以直接点击“写日记”。

### 2.2 文件夹详情页（层级递进）
- **定位**：用户点击某个具体的文件夹后进入的页面。
- **展示内容**：
  - 明确的面包屑导航提示（如：首页 > 学习 > 日语），告诉用户当前身处何处。
  - 当前文件夹内的子文件夹和具体的日记列表。
- **主要操作**：点击任意日记进入阅读/编辑页，或者点击返回键回到上一级。

### 2.3 日记编辑与阅读页
- **定位**：全屏的沉浸式页面，用于阅读过去的日记或撰写新日记。
- **展示内容**：日记的标题、时间、心情/天气、正文内容以及穿插的图片。
- **主要操作**：编辑内容，插入照片，修改背景，保存退出。

### 2.4 设置与数据管理中心
- **定位**：处理与日记内容无关的底层配置功能。
- **主要操作**：
  - **回收站入口**：查看、恢复或彻底清除被删掉的文件夹和日记。
  - **数据备份与恢复**：执行 `.diary` 文件的导出或导入。
  - **主题与隐私保护**：开关指纹锁/密码锁功能，切换 App 的外观风格（如夜间模式）。
